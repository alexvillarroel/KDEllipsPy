NA_kine


                                               Using  NA_kine

The use of this program is pretty complex because the philosophy of axitra and NA_kine are pretty different. We modified convm separating the reading of Green's functions from convolution but there are still "funny" things like the axi.dat file.

Procedure to model an earthquake

1. Seismic Data

The first thing is to get seismograms digitized at constant interval and put one after the other in a single ascii file. The procedure to follow to convert accelrograms, filter and intergate them with SAC and then to produce ascii data may be streamlined with Python.

Number of points: because axitra uses FFT the number of points in each component must be a power of 2 (512 is pretty good). The time step must be adjusted so that the seismograms of all the stations enter within the window. For Tocopilla tipically use:

Delta t = 0.2 , Npts=512, Ttotal time=10.24

In order to invert synthetic seismograms for testing, etc. One can generate a model using  na_kine_direct that uses the same data. The observed seismograms in that case may be an empty array, but it is better to use actual records.  

2. Input to axitra

You need to collect information about the earthquake and the stations used in several files. We provice a program stations.f that produces both the station and stationn files

2. a) Station and stationn files contain position and names of the stations to be used. The coordinates are in km wrt to the coordinate system  used by axitra. Normally it is

y East,  x North, z Up.

Note that this is not  a right hand side coordinate system.

In order to generate station and stationn you can use the program

stations <stations.in

where stations.in contains the source coordinates,
the number of stations,
geographical coordinates  (latitude and longitude) for each   station

2. b) Fault The program  faille produces the most essential input to axitra, the source.dat file that defines all the point sources for which a Green function will be generated.


The program faille is run by

faille

it reads the description of the fault from file

IN/ faille.in

Example of faille.in:
144000 48000                 % longueur(stk) et largeur(dip) en m
100000 34000                     % position de l'epicentre sur la faille en m
45000                                % profondeur de l'hypocentre en m
30 10                                      % nb de sousfailles selon le strke et le dip
10                                         % azimut de la faille en degre parrapport au nord
30                                       % pendage (dip) en degre
85                                % rake en degre
3                                % amplitude du glissement (m)
2800                                % vitesse de rupture en m/s

Definition of these variables is in the Figure 1 at the bottom

The size of the fault is quite clear
The position of epicentre is that of the upper left corner of the fault wrt to the Hypocenter defined in stations.in. That is the origin (0,0,0) point of the system of coordinates.

The most important parameters are the fault angles. These follow Aki and Richard's convention:
azimut   is the angle of the strike of the fault wrt North
dip                is the dip angle of the fault wrt the surface of the earth
rake        is the angle of the slip vector wrt the horizontal along the fault

Slip and rupture velocity are obvious.


Faille produces the file source.dat, the main input to axitra

Changes in faille.in are VERY IMPORTANT


You can PLOT the model and stations using

stations.py
produces two nice plots of the data input to axitra. You can then modify the faille.in file and see how does the fault move wrt to the stations.

Once the files station, stationn and source.dat are ready we can run axitra

This is a weird file with an input "namelist" and a velocity model

The namelist contains

NC number of layers in the model
nfreq number of frequencies to be used in the synthetics (128)
t     duration of the seismogram
aw  Bouchon's funny parameter use 2
nr    number of stations
ns   number of fault elements (equal to those defined in faille.in)
XL  Bouchon's idem
t0=rise time
fr1=0.05  low frequency cut
fr2 = 1     high frequency cut
ikmax Bouchon's idem
uconv  idem

sourcefile = "source.dat"
statfile='station'

Axitra produces two files:

axi.hist   description of the fault with  rather useless details
axi.res    the Green functions in binary format

4. Kinematic modelling

This is done by the program

../bin/kine_direct

For historical reasons input to this program is very complex, but it reads just the files:

axi.dat                                                        already described in axitra and
kine_files/kine.in                                most of this file still unused

In kine.in check only
nsource=300 size of source file
xhypo yhypo nstr ,dip, n_ellipse and n sismograms
time windows and ndata = 4*128=512

4. Output

There are several output files. We follow the traditional outputs of Sambridge:

na.det    detailed results for each iteration
na.sum  summary file (completely useless part of NA.F)
na.nad   binary file with models and misfits.  

and we add

kine_files/models.dat  ascii list of all models computed
kine_files/misfit.dat   ascii list of all misfits
kine_files/model_best.dat    best model
kine_files/slip_fin                  slip for best model
kine_files/tR_FIN                  rupture time for best model
